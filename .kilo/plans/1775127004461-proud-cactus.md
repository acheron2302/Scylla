# Plan: Add Image Base Choice Option for IAT Fix

## Goal
Add a GUI option allowing users to choose which image base to use when fixing IAT.

## Current Behavior
- When fixing IAT, `ImportRebuilder` uses the image base from the PE header
- `PeRebuild::reBasePEImage()` exists but is not integrated into the IAT fix flow
- No way for users to specify a custom image base

## Implementation Steps

### 1. Add Configuration Option
**File: `src/ConfigurationHolder.h`**
- Add `CUSTOM_IMAGE_BASE` to `ConfigOption` enum
- No changes needed to `ConfigurationHolder.cpp` since it handles numeric values generically

### 2. Add Method to PeParser for Image Base
**File: `src/PeParser.h`**
- Add `void setImageBase(DWORD_PTR newImageBase);` method declaration

**File: `src/PeParser.cpp`**
- Add `setImageBase()` implementation that updates `moduleBaseAddress` and the PE header's `OptionalHeader.ImageBase`

### 3. Integrate Rebasing into ImportRebuilder
**File: `src/ImportRebuilder.h`**
- Add `DWORD_PTR customImageBase;` member variable
- Add `bool useCustomImageBase;` flag
- Add `void setCustomImageBase(DWORD_PTR newImageBase);` method

**File: `src/ImportRebuilder.cpp`**
- Initialize `customImageBase = 0` and `useCustomImageBase = false` in constructor
- In `rebuildImportTable()` or `buildNewImportTable()`, after `isValidPeFile()` and `readPeSectionsFromFile()`:
  - If `useCustomImageBase` is true, call `PeRebuild::reBasePEImage()` with `customImageBase`
  - This requires `ImportRebuilder` to have access to `PeRebuild::reBasePEImage()`

**Alternative approach (simpler):**
- Add a public method `reBaseImage(DWORD_PTR newBase)` in `ImportRebuilder` that calls `PeRebuild::reBasePEImage()`

### 4. Update Options GUI
**File: `src/OptionsGui.h`**
- Add `bool useCustomImageBase;` member
- Add `DWORD_PTR customImageBase;` member  
- Add `CEdit EditCustomImageBase;` control
- Add DDX mapping for new controls
- Add checkbox to enable custom image base

**File: `src/OptionsGui.cpp`**
- Add ID definitions for new controls in `resource.h` (e.g., `IDC_CHECK_CUSTOM_IMAGE_BASE`, `IDC_EDIT_CUSTOM_IMAGE_BASE`)
- Implement `loadOptions()` and `saveOptions()` for the new settings

### 5. Update Resource File
**File: `src/MainGui.rc`**
- Add checkbox control: "Use custom image base"
- Add edit control for hex image base value

### 6. Connect Options to MainGui
**File: `src/MainGui.cpp`**
- In `dumpFixActionHandler()`, after creating `ImportRebuild`:
  - Check if `Scylla::config[CUSTOM_IMAGE_BASE]` is enabled
  - If yes, call `importRebuild.setCustomImageBase(value)`

### 7. Add Control IDs
**File: `src/resource.h`**
- Add `#define IDC_CHECK_CUSTOM_IMAGE_BASE 1400` (or next available ID)
- Add `#define IDC_EDIT_CUSTOM_IMAGE_BASE 1401`

## Files to Modify
1. `src/ConfigurationHolder.h` - Add config enum
2. `src/PeParser.h` - Add setImageBase method
3. `src/PeParser.cpp` - Implement setImageBase
4. `src/ImportRebuilder.h` - Add custom image base members and methods
5. `src/ImportRebuilder.cpp` - Implement rebasing logic
6. `src/OptionsGui.h` - Add GUI members
7. `src/OptionsGui.cpp` - Implement load/save
8. `src/MainGui.cpp` - Connect options to ImportRebuilder
9. `src/resource.h` - Add control IDs
10. `src/MainGui.rc` - Add UI controls

## Notes
- The `PeRebuild::reBasePEImage()` requires the new base to be aligned to 0x10000
- Should validate the custom image base before calling reBase
- Should log errors if rebasing fails
